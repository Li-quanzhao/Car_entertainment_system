# GitHub 推送规则

> 远程仓库 `origin` 已配置为 `https://github.com/Li-quanzhao/Car_entertainment_system.git`

---

## 自动推送触发条件

满足以下**任一条件**时，应在任务完成后主动询问用户是否推送到 GitHub：

| 触发条件 | 示例 |
|---------|------|
| **新增文件 ≥ 2 个** | 新增 ViewModel + Service 一组文件 |
| **修改文件 ≥ 4 个** | 跨层修改涉及 Infra/Service/ViewModel/QML |
| **新增重要功能** | 新增一个完整页面、新增一个 Service 模块、新增 Agent 工具 |
| **完成一个 6A 子任务** | TASK 文档中标记 ✅ 的原子任务 |
| **修复关键 Bug** | 编译错误修复、运行时崩溃修复、数据错误修复 |
| **用户显式要求** | `/push` 命令或 "推送" / "提交" |

## 不触发推送的情况

以下情况**不需要**推送：
- 修改 1-2 个文件的小调整（颜色、文字、注释）
- 临时调试代码（未完成的功能）
- 仅修改文档（不涉及源代码变更）
- 用户明确说"先不推送"

## 推送流程

1. **判断触发条件** — 任务完成后检查是否满足上述条件
2. **询问用户** — 列出变更摘要，询问是否推送
3. **用户确认后执行**:
   - `git status` 检查当前状态
   - `git add -A` 暂存所有更改
   - `git commit -m "..."` 提交（提交信息需用户确认）
   - `git push origin` 推送到远程

## 推送询问格式

```
[推送建议] 本次修改:
  - 新增: xxx_viewmodel.h, xxx_viewmodel.cpp (2 文件)
  - 修改: CMakeLists.txt, main.cpp (2 文件)
  - 原因: 新增 XXX 功能模块
  是否推送到 GitHub? (y/n)
```

> **重要**: 绝不自动推送，必须先询问用户确认。`push --force` 等破坏性操作绝对禁止（详见 AI自我治理规则 6.3）。
