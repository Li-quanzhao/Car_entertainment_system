## 构建流程

### HMI (C++ Qt)

> **重要**: 项目路径 `Trae实验` 含中文，MinGW Make 通过 cmd.exe 解码会乱码，需要用目录链接：
> ```powershell
> New-Item -ItemType Junction -Path "E:\car_hmi_project" -Target "E:\PyCharmProject\Trae实验\Car_entertainment_system" -Force
> ```

```bash
cd /d E:\car_hmi_project\hmi
# 首次配置
cmake -B build -G "MinGW Makefiles" -DCMAKE_PREFIX_PATH=E:/Qt/6.11.1/mingw_64
# 编译
cmake --build build
# 运行
./build/car_hmi.exe
```

> 每次修改 C++ 代码后需要重新 `cmake --build build`。修改 QML 文件后如果 qml.qrc 未变则无需重新编译。预生成的 MOC 文件 (`moc_serial_port_adapter.cpp`, `moc_bluetooth_adapter.cpp`) 在 `.gitignore` 中，新增 Q_OBJECT 类后需手动运行 moc：
> ```bash
> E:\Qt\6.11.1\mingw_64\bin\moc.exe src/infrastructure/xxx.h -o src/infrastructure/moc_xxx.cpp
> ```

### Agent (Python)

cd agent

## 已知限制
1. **Ninja 中文路径编码问题** — 项目路径 `Trae实验` 含中文，Ninja 输出的路径字符串被沙箱错误编码，导致找不到文件。**解决方案**: 使用 `MinGW Makefiles` 生成器。
2. **Qt6 缺失 SerialPort/Bluetooth** — 使用 Win32 API 手写补齐，实现位于 `hmi/src/infrastructure/`。
3. **gRPC 当前未启用** — 通信使用 HTTP/JSON，gRPC 为后续升级目标。`protoc` 已下载至 `E:\protoc\bin`。
4. **CMake AUTOMOC 必须开启** — `serial_port_adapter.h` 和  `bluetooth_adapter.h` 均包含 `Q_OBJECT` 宏，需 `CMAKE_AUTOMOC ON`。
5. **所有 ViewModel 需手动注册** — 新 ViewModel 需在 `main.cpp` 调用 `qmlRegisterType<T>()`，QML 才能识别。
6. **BOM 编码** — 推荐所有源文件使用 UTF-8 with BOM 编码，避免 MinGW 中文注释乱码。
