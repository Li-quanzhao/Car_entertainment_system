## 已知限制
1. **gRPC C++ → MinGW** — Anaconda gRPC CMake 配置深度注入 MSVC 链接标志（`-ignore:4221`），MinGW/g++ 无法消化。**已解决**: 切换到 MSVC 工具链。
2. **gRPC 双协议已启用** — Agent 同时运行 HTTP :8000 + gRPC :50051，HMI C++ 通过 MSVC 同时支持 HTTP + gRPC。**已部署**: Windows 服务 `CarAgentServer` 开机自启。
3. **Ninja 中文路径编码问题** — MinGW 专用问题。MSVC 支持 UTF-8 路径无需处理。
4. **Qt6 缺失 SerialPort/Bluetooth** — 使用 Win32 API 手写补齐，实现位于 `hmi/src/infrastructure/`。
5. **CMake AUTOMOC 必须开启** — `serial_port_adapter.h` 和 `bluetooth_adapter.h` 均包含 `Q_OBJECT` 宏，需 `CMAKE_AUTOMOC ON`。
6. **所有 ViewModel 需手动注册** — 新 ViewModel 需在 `main.cpp` 调用 `qmlRegisterType<T>()`，QML 才能识别。
7. **BOM 编码** — 推荐所有源文件使用 UTF-8 with BOM 编码，避免 MinGW 中文注释乱码。
8. **Anaconda protobuf 版本号异常** — `Anaconda/Library/lib/python3.13/site-packages/google/protobuf/runtime_version.py` 中 `PROTOBUF_VERSION` 被设为过期值 `5.29.3`，导致 gRPC proto 导入失败。需手动改为 `6.33.6`。
9. **grpcio 与 LocalSystem 账户** — 用 pip 用户安装的 grpcio 对 LocalSystem (Windows 服务) 不可见，必须安装到项目 venv (`--target`)。
